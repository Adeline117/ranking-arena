# ─────────────────────────────────────────────────────────────
# .tsc-legacy-errors.txt
#
# Regex patterns (one per line, extended regex / `grep -E`) that mark
# TypeScript errors as LEGACY — known-broken files that we do not want
# to block pushes.
#
# Lines starting with `#` and blank lines are ignored by the loader in
# `.git/hooks/pre-push`.
#
# Rules for adding entries here:
#   1. Be SPECIFIC — file paths are preferred over error messages.
#      Using broad error-message patterns (e.g. "implicitly has an .any.
#      type") masks genuine regressions in new code.
#   2. Include a one-line rationale above each block so future readers
#      know why an entry exists and can remove it once fixed.
#   3. When a file is fixed, DELETE its entry immediately. Leaving
#      stale entries defeats the allow-list.
#
# This file replaces the ~120-entry inline `grep -v` chain that used to
# live in the pre-push hook. See the git history on .git/hooks/pre-push
# for the original.
# ─────────────────────────────────────────────────────────────
