## 一、用户主页模块总览

豆瓣用户主页采用**左右两栏布局**，左侧为用户信息侧边栏，右侧为内容区域，各模块按类别垂直排列。

### 1. 用户信息区（侧边栏）
- **头像 + 昵称**
- **个性签名**（如 "less is more"）
- **常居城市**
- **用户ID**（数字ID + 自定义URL别名）
- **注册日期**
- **操作按钮**：关注此人、发豆邮

### 2. 广播（动态 Feed）
- 模块标题：「XX的广播」
- 链接到完整动态列表 `/people/{id}/statuses`
- 未登录状态下不可查看详细内容
- **特点**：动态是用户行为的聚合，包括标记书影音、写评论、转发等

### 3. 书（读书）
- 模块标题：「XX的书」
- **三种状态标签**：在读 / 想读 / 读过（各显示数量）
- **书单**：用户创建的主题书单
- **杂志订阅**
- 每种状态默认显示5本书的封面缩略图
- 链接结构：`book.douban.com/people/{id}/do|wish|collect`

### 4. 电影
- 模块标题：「XX的电影」
- **三种状态标签**：在看 / 想看 / 看过（各显示数量）
- **片单**：用户创建的电影片单
- 每种状态默认显示5部电影海报缩略图
- 链接结构：`movie.douban.com/people/{id}/do|wish|collect`

### 5. 音乐
- 模块标题：「XX的音乐」
- **三种状态标签**：在听 / 想听 / 听过
- 每种状态默认显示5张专辑封面

### 6. 游戏
- 模块标题：「XX的游戏」
- 状态：想玩 / 玩过

### 7. 评论（书评/影评）
- 模块标题：「XX的评论」
- 显示评论总数
- 列表展示：评论标题 + 所评对象 + 评论摘要 + 回应数

### 8. 同城活动
- 参加 / 感兴趣 两种状态
- 显示活动名称、时间、地点、关注人数

### 9. 线上活动
- 标签、发起人、参加人数

### 10. 关注的小站
- 小站名称 + 图标

### 11. 移动应用
- 用过的app

### 12. 其他入口
- 去过的地方（旅行）
- 添加的条目
- RSS订阅（收藏feed）

## 二、"想读/在读/读过" 功能流程

### 标记流程
1. 用户在**书籍详情页**点击「想读」「在读」「读过」按钮
2. 标记时可选择：
   - ⭐ **评分**（1-5星，对应2-10分）
   - ✏️ **短评**（一句话评价）
   - 📅 **标记日期**
   - 🏷️ **标签**（自定义标签，如"科幻"、"经典"）
3. 标记后自动归入用户主页对应分类
4. 状态可随时更新（想读→在读→读过）

### 数据展示
- 用户主页显示各状态数量：「60本在读 · 81本想读 · 115本读过」
- 点击进入完整列表，支持按时间/评分排序

## 三、评分 + 短评 + 长评体系

### 1. 评分系统
- **5星制**（对应10分制）
- 每颗星对应：⭐很差 ⭐⭐较差 ⭐⭐⭐还行 ⭐⭐⭐⭐推荐 ⭐⭐⭐⭐⭐力荐
- 评分在标记"读过/看过"时填写
- 条目页面展示**加权平均分**和**评分分布图**（柱状图）

### 2. 短评
- 标记时一并填写，字数限制较短（约350字）
- 按「最新」「热门」排序
- 支持"有用"点赞
- 显示评价者的评分星级

### 3. 长评（书评/影评）
- 独立的评论文章，支持富文本
- 有标题，支持长篇幅
- 可被评论、点赞、转发
- 在条目详情页展示：标题 + 作者 + 发布日期 + 摘要
- 书评总数显示（如"全部5044条"）
- **热门评论**优先展示

### 4. 原文摘录
- 用户可提交书中原文段落
- 独立模块，支持浏览和引用

## 四、动态 Feed 展示

### Feed 内容类型
- 标记书影音（想读/在读/读过）
- 发表短评/长评
- 创建书单/片单
- 转发他人内容
- 发表日记
- 参加活动

### 展示方式
- 时间线倒序排列
- 每条动态包含：用户头像 + 行为描述 + 内容预览 + 时间戳
- 支持评论、转发、点赞互动
- 未登录用户无法查看

### Feed URL
- `/people/{id}/statuses` — 完整动态列表

## 五、条目详情页结构（以书籍为例）

以《红楼梦》为例：

1. **基本信息区**：书名、作者、出版社、页数、定价、ISBN
2. **评分区**：综合评分 + 评分人数 + 评分分布图
3. **操作按钮**：想读 / 在读 / 读过
4. **内容简介**：可展开/收起
5. **作者简介**
6. **目录**：可展开
7. **原文摘录**：精选段落
8. **丛书信息**
9. **推荐**："喜欢这本书的人也喜欢"
10. **书评区**：热门长评列表（标题 + 作者 + 日期 + 摘要 + 展开）
11. **短评区**
12. **读书笔记**
13. **论坛讨论**
14. **电子书购买入口**

## 六、可借鉴的功能点

### 🎯 核心交互模式
| 功能 | 豆瓣实现 | 借鉴价值 |
|------|----------|----------|
| 三态标记 | 想读/在读/读过 | ⭐⭐⭐⭐⭐ 极简且直觉的状态管理 |
| 评分体系 | 5星制 + 分布图 | ⭐⭐⭐⭐⭐ 既简单又有数据可视化 |
| 短评+长评分离 | 标记时短评，独立写长评 | ⭐⭐⭐⭐ 降低评价门槛 |
| 用户档案 | 书影音分类展示 | ⭐⭐⭐⭐ 身份认同和展示需求 |
| 标签系统 | 用户自定义标签 | ⭐⭐⭐⭐ 灵活分类 |
| 豆列（书单/片单）| 用户策展 | ⭐⭐⭐⭐ UGC策展内容 |
| 动态Feed | 行为聚合时间线 | ⭐⭐⭐ 社交发现 |

### 💡 设计理念
1. **低门槛参与**：一键标记 → 可选评分 → 可选短评 → 可选长评，逐层递进
2. **数据即内容**：用户的标记行为本身就构成内容（动态、统计）
3. **社区自组织**：标签、书单、小组都是用户驱动的
4. **个人档案=身份**：你读过什么、看过什么，定义了你是谁

## 七、与 Ranking Arena 的映射关系

| 豆瓣概念 | Ranking Arena 对应 | 说明 |
|----------|-------------------|------|
| 书 / 电影 / 音乐 | **Library 内容条目** | 我们的"条目"是交易员、策略、交易所等 |
| 想读 / 在读 / 读过 | **关注 / 跟踪中 / 已评价** | 用户对交易员/策略的关注状态 |
| 5星评分 | **排名/评分系统** | 用户对交易员的评价 |
| 短评 | **一句话点评** | 对交易员/策略的简短评价 |
| 长评（书评/影评）| **深度分析文章** | 对交易员/策略的详细分析报告 |
| 书单 / 片单 | **自选榜单** | 用户策展的"我推荐的交易员"列表 |
| 标签 | **标签系统** | 如"量化"、"日内"、"趋势跟踪"等 |
| 用户主页 | **个人档案页** | 展示用户的关注、评价、榜单 |
| 动态 Feed | **社区动态** | 用户的评价、关注行为聚合 |
| 评分分布图 | **评分可视化** | 交易员的评价分布 |
| 小组 | **讨论社区** | 按主题的讨论区 |
| 原文摘录 | **数据摘录/截图** | 交易员的关键数据点引用 |
| "喜欢XX的人也喜欢" | **推荐系统** | 协同过滤推荐 |

### 具体映射示例

```
豆瓣：用户A 标记《红楼梦》为"读过" ⭐⭐⭐⭐⭐ "不朽的经典"
我们：用户A 标记 "交易员张三" 为"已评价" ⭐⭐⭐⭐⭐ "稳定盈利，回撤控制好"

豆瓣：用户A的书架 — 60本在读 · 81本想读 · 115本读过
我们：用户A的关注 — 12个跟踪中 · 25个关注 · 48个已评价

豆瓣：红楼梦 — 评分9.6 · 5044条书评
我们：交易员张三 — 评分4.8 · 128条评价
```

## 九、设计建议

1. **优先实现三态标记系统**：这是豆瓣最核心的交互，简单有效
2. **评分采用5星制**：比10分制更直觉，但后端可存为10分制精度
3. **短评零门槛**：标记时顺手写，不强制
4. **个人档案页要好看**：这是用户留存的关键，要有"成就感"
5. **Feed不急做**：先做好条目和评价，再做社交层
