# 响应式设计说明

本文档说明 Ranking Arena 的响应式设计实现，确保网站在不同设备上都能提供良好的用户体验。

## 设备支持

### 断点定义

- **手机（Mobile）**: < 640px
- **平板（Tablet）**: 640px - 1024px
- **桌面（Desktop）**: 1024px - 1440px
- **大屏（Large Desktop）**: > 1440px

### 支持的设备类型

- ✅ iPhone（各种型号，包括 iPhone X 系列）
- ✅ Android 手机（各种尺寸）
- ✅ iPad（竖屏和横屏）
- ✅ Android 平板
- ✅ 笔记本电脑（13" - 15"）
- ✅ 桌面显示器（> 15"）

## 响应式布局

### 主页面布局

主页面使用三栏布局，在不同设备上自动调整：

#### 桌面（> 1024px）
```
[热门讨论] [排行榜] [市场面板]
  320px      1fr      280px
```

#### 平板（768px - 1024px）
```
[热门讨论] [排行榜] [市场面板]
  280px      1fr      240px
```

#### 手机和平板竖屏（< 768px）
```
[排行榜]
  1fr

[热门讨论]
  1fr

[市场面板]
  1fr
```

### 排行榜表格

排行榜表格在不同设备上显示不同的列：

#### 桌面（> 1024px）
显示所有列：排名、交易员、ROI、胜率、交易量、平均买入

#### 平板（768px - 1024px）
显示：排名、交易员、ROI、胜率、交易量、平均买入（列宽调整）

#### 手机横屏（640px - 768px）
显示：排名、交易员、ROI、胜率（隐藏交易量和平均买入）

#### 手机竖屏（< 640px）
显示：排名、交易员、ROI（仅显示核心信息）

## 顶部导航

### 桌面和平板
- 显示完整导航：Logo、导航链接、搜索框、用户菜单
- 搜索框最大宽度：520px

### 手机
- 隐藏搜索框（可通过其他方式访问）
- 简化导航链接
- 优化触摸目标大小（最小 44x44px）

## 登录页面

### 桌面和平板
- 居中显示，最大宽度 420px
- 标准内边距：32px

### 手机
- 全宽显示，边距 16px
- 内边距：24px 20px
- 输入框字体大小：16px（防止 iOS 自动缩放）
- 按钮最小高度：44px

## 触摸优化

### 触摸目标大小
- 所有可点击元素最小尺寸：44x44px（符合 iOS 和 Android 指南）
- 按钮内边距：12px 16px
- 图标按钮：12px 内边距

### 触摸反馈
- 所有交互元素都有视觉反馈
- 使用 CSS transition 提供平滑动画

## 字体大小

### 桌面
- 基础字体：14px
- 标题 H1：24px
- 标题 H2：20px
- 标题 H3：18px

### 平板
- 基础字体：13px
- 标题相应缩小

### 手机
- 基础字体：13px
- 标题 H1：20px
- 标题 H2：18px
- 标题 H3：16px
- 输入框：16px（防止 iOS 自动缩放）

## 安全区域适配

针对 iPhone X 及更新型号，支持安全区域：

```css
.safe-area-top {
  padding-top: max(12px, env(safe-area-inset-top));
}

.safe-area-bottom {
  padding-bottom: max(12px, env(safe-area-inset-bottom));
}
```

## 性能优化

### 图片优化
- 使用响应式图片（如需要）
- 懒加载非关键图片

### CSS 优化
- 使用 CSS Grid 和 Flexbox（现代浏览器原生支持）
- 避免使用 JavaScript 进行布局计算
- 使用 CSS 变量实现主题切换

### 设备测试清单

- [ ] iPhone SE（小屏手机）
- [ ] iPhone 12/13/14（标准手机）
- [ ] iPhone 14 Pro Max（大屏手机）
- [ ] iPad（竖屏和横屏）

### 测试要点

1. **布局检查**
   - 确保所有内容在视口内可见
   - 没有水平滚动条
   - 元素不会重叠

2. **交互检查**
   - 所有按钮和链接都可以点击
   - 触摸目标足够大
   - 表单输入正常工作

3. **性能检查**
   - 页面加载速度
   - 滚动流畅度
   - 动画性能

4. **可访问性检查**
   - 文字大小可读
   - 颜色对比度足够
   - 键盘导航支持

## 常见问题

### Q: 为什么输入框字体大小设置为 16px？

A: iOS Safari 在输入框字体小于 16px 时会自动缩放页面，导致用户体验不佳。设置为 16px 可以防止自动缩放。

### Q: 如何测试不同设备？

A: 
1. 使用浏览器开发者工具的设备模拟器
2. 使用真实设备测试
3. 使用在线工具如 BrowserStack

### Q: 响应式设计会影响性能吗？

A: 不会。我们使用 CSS 媒体查询，这是浏览器原生支持的功能，性能开销极小。
