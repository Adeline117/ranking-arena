# PM-20260422：629 个 commit 无人验证，交易员详情页 500 持续数天

| 字段     | 值                                                                   |
| -------- | -------------------------------------------------------------------- |
| 日期     | 2026-04-22（发现日）；影响窗口约数天（准确起点未知——这本身就是问题） |
| 严重度   | SEV1（核心路径：所有交易员详情页崩溃）                               |
| 用户影响 | 全部用户无法打开任何交易员详情页（核心路径第一环节）                 |
| 检测方式 | 事后人工审计发现，非告警——累积 3 个 BLOCKER + 8 个 HIGH 无人知晓     |

## 时间线

- ~2026-04 中旬起：连续高速迭代，629 个 commit 直推 main 并自动部署，无部署后验证
- 2026-04-22：人工审计发现交易员详情页全量 500，已持续数天

## 根因

直接原因：某次改动破坏交易员详情页渲染 → 为什么没发现：部署后无任何自动验证
→ 为什么没有验证：Vercel push 即部署，流程里没有"部署后"这个环节
→ 系统性根因：**AI 生成速度远超验证速度，且整条链路（commit→push→deploy→线上）
没有任何一个强制核验点**。改了代码没人看线上。

## 修复

- 止血：Vercel Dashboard 回滚到上一个绿色部署
- 根治：`scripts/post-deploy-check.sh`（5 条核心 URL 部署后核验）+
  `.github/workflows/post-deploy-smoke.yml`（deployment_status 自动触发）+
  写路径金丝雀（发帖→赞→评论→删）

      push 不再直通生产，[deploy-force] 逃生口，详见 RUNBOOK「部署管线」）

## 教训

会爆炸的系统会被立刻修；安静失败的系统积债到用户撞见。任何"部署"动作后面
必须跟一个不依赖人自觉的核验机制。
