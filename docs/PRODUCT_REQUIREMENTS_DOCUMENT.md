## 文档信息
- **产品名称**: Ranking Arena
- **版本**: v1.0
- **创建日期**: 2026-02-06
- **产品定位**: 加密货币交易员排行榜与社区平台

---

## 1. 产品概述

### 1.1 产品简介
Ranking Arena 是一个聚合多个加密货币交易所（Binance、Bybit、Bitget、MEXC、OKX、KuCoin、CoinEx、GMX）跟单交易数据的透明化交易员排行榜与社区平台。平台通过数据抓取、分析和社区互动，帮助用户发现优质交易员并进行社交讨论。

### 1.2 核心价值
- **透明化**: 聚合多交易所数据，提供客观的交易员排名
- **社区化**: 用户可以讨论、分享交易策略和观点
- **便捷性**: 一站式查看多个交易所的交易员表现
- **实时性**: 定期更新交易员数据，保持信息新鲜度

### 1.3 目标用户
- **普通投资者**: 寻找优质交易员进行跟单
- **交易员**: 展示交易能力，吸引跟单者
- **社区用户**: 讨论交易策略，分享市场见解
- **数据分析师**: 研究交易数据和市场趋势

## 3. 核心功能模块

### 3.1 排行榜模块

#### 3.1.1 排行榜首页
**页面路径**: `/` 或 `/rankings`

**页面元素**:
- 顶部导航栏（Logo、搜索框、用户菜单）
- 筛选器面板
- 排序控制器
- 交易员列表（表格/卡片视图）
- 分页/无限滚动

**用户操作流程**:

```
用户进入首页
    ↓
查看默认排行榜（按 Arena Score 排序）
    ↓
[可选] 使用筛选器
    ├─ 选择交易所（Binance/Bybit/等）
    ├─ 选择时间范围（7天/30天/90天/全部）
    ├─ 设置 ROI 范围（最小-最大）
    ├─ 设置最大回撤范围
    ├─ 设置 Arena Score 范围
    └─ 设置最小跟单人数/AUM
    ↓
[可选] 更改排序方式
    ├─ Arena Score（默认）
    ├─ ROI（收益率）
    ├─ 回撤率
    ├─ 跟单人数
    └─ AUM（管理资金）
    ↓
[可选] 切换视图模式
    ├─ 表格视图（详细）
    └─ 卡片视图（精简）
    ↓
浏览交易员列表
    ↓
点击交易员 → 进入交易员详情页
```

**筛选器详细规格**:

| 筛选器 | 类型 | 选项 | 默认值 |
|--------|------|------|--------|
| 交易所 | 多选 | Binance, Bybit, Bitget, MEXC, OKX, KuCoin, CoinEx, GMX | 全部 |
| 时间范围 | 单选 | 7天, 30天, 90天, 全部 | 30天 |
| ROI 范围 | 滑块 | 0% - 1000%+ | 全部 |
| 最大回撤 | 滑块 | 0% - 100% | 全部 |
| Arena Score | 滑块 | 0 - 100 | 全部 |
| 最小跟单人数 | 输入框 | 0+ | 0 |
| 最小 AUM | 输入框 | $0+ | 0 |

**交互细节**:
1. 筛选条件变化时，URL 参数同步更新（便于分享）
2. 登录用户的筛选偏好自动保存到 localStorage
3. 表格支持列排序、列宽调整、列显示/隐藏
4. 表格第一列（排名/交易员）固定不滚动
5. 表格行 hover 时高亮显示
6. 卡片视图支持响应式布局（桌面 3列，平板 2列，手机 1列）
7. 无限滚动触底时加载下一页（50条/页）
8. 骨架屏加载动画

**关键指标展示**:
- **排名**: 序号 + 趋势箭头（↑↓）
- **交易员信息**: 头像 + 昵称 + 交易所标识
- **ROI**: 百分比 + 颜色编码（绿涨红跌）
- **最大回撤**: 百分比 + 警告提示（>30% 标红）
- **Arena Score**: 评分 + 星级显示
- **跟单人数**: 人数 + 趋势
- **AUM**: 金额格式化（$1.2M）

#### 3.1.2 表格/卡片视图

  - 响应式尺寸

#### 4.1.2 数据加载优化
- **SWR**: 客户端数据获取 + 缓存
  - `revalidateOnFocus`: 页面重新获焦时刷新
  - `revalidateOnReconnect`: 网络重连时刷新
  - `dedupingInterval`: 去重间隔 2000ms
- **Redis 缓存**:
  - 排行榜数据缓存 5 分钟
  - 交易员详情缓存 15 分钟
  - 搜索结果缓存 10 分钟
- **数据预获取**:
  - Link hover 时预获取目标页面数据
  - 无限滚动提前加载下一页

#### 4.1.3 渲染优化
- **虚拟滚动**: 长列表（>100项）使用虚拟滚动
- **React.memo**: 避免不必要的重渲染
- **useMemo/useCallback**: 优化计算和函数引用

### 4.2 实时功能

#### 4.2.1 Supabase Realtime
**使用场景**:

- 在线状态同步

**实现示例**:
```typescript
// 监听关注交易员的数据更新
const subscription = supabase
  .channel('trader-updates')
  .on(
    'postgres_changes',
    {
      event: 'UPDATE',
      schema: 'public',
      table: 'traders',
      filter: `id=in.(${followedTraderIds.join(',')})`,
    },
    (payload) => {
      // 更新本地数据
      updateTraderData(payload.new)
    }
  )
  .subscribe()
```

### 4.3 数据抓取系统

#### 4.3.1 架构设计
```
Vercel Cron Jobs (调度器)
    ↓
触发 API 端点
    ↓
Worker Service (独立服务)
    ├─ Puppeteer Cluster (并发爬虫)
    ├─ 代理池管理
    ├─ 请求队列
    └─ 数据验证
    ↓
数据存储
    ├─ Supabase (结构化数据)
    └─ Redis (缓存)
```

#### 4.3.2 抓取策略
**热门交易员** (Top 100):
- 频率: 每15分钟
- 优先级: 高
- 数据: 基础信息 + 实时持仓

**活跃交易员** (Top 1000):
- 频率: 每小时
- 优先级: 中
- 数据: 基础信息

**普通交易员**:
- 频率: 每4小时
- 优先级: 低
- 数据: 基础信息（按需）

#### 4.3.3 反反爬虫措施
- 代理 IP 池轮换
- User-Agent 随机化
- 请求频率控制（每个交易所独立限流）
- 随机延迟（2-5秒）
- 模拟真实浏览器行为（滚动、点击等）
- Cookie 管理

### 4.4 安全措施

#### 4.4.1 认证与授权
- **JWT**: Supabase Auth + JWT
- **RLS**: Supabase Row Level Security
- **CSRF**: CSRF Token 保护
- **XSS**: 内容过滤 + CSP 策略

#### 4.4.2 Rate Limiting
```
API 端点限流:
- 未认证: 10 req/min
- 已认证: 60 req/min
- Premium: 120 req/min
```

#### 4.4.3 内容安全
- **输入验证**: Zod schema 验证
- **SQL 注入**: Supabase 参数化查询
- **XSS 防护**: DOMPurify 净化 HTML
- **CORS**: 限制允许的来源
