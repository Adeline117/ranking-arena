# Arena Score 算法说明文档

## 概述

Arena Score 是 Arena 平台用于评估和排名加密货币交易员的综合评分系统。该系统旨在平衡收益能力和风险控制，为用户提供一个更全面的交易员评估指标。

## 设计原则

1. **收益导向**：收益仍然是最重要的指标，占总分的 85%
2. **风险意识**：通过回撤和稳定性指标惩罚高风险交易者
3. **时间适应**：不同时间周期使用不同的参数配置
4. **门槛筛选**：设置 PnL 门槛过滤低交易量账户

## 评分结构

Arena Score 由三个部分组成：

| 组成部分 | 权重 | 说明 |
|---------|------|------|
| 收益分 (Return Score) | 0-85 分 | 基于 ROI 的收益评分 |
| 回撤分 (Drawdown Score) | 0-8 分 | 基于最大回撤的风险评分 |
| 稳定分 (Stability Score) | 0-7 分 | 基于胜率的稳定性评分 |

**总分 = 收益分 + 回撤分 + 稳定分（0-100 分）**

## 详细计算方法

### 1. 收益分 (0-85 分)

收益分使用年化收益强度和双曲正切函数计算，目的是：
- 奖励高收益交易员
- 压缩极端高收益的分数差距（避免少数人垄断榜首）

**计算公式：**

```
收益强度 I = (365 / 天数) × ln(1 + ROI)
基础分 R₀ = tanh(系数 × I)
收益分 = 85 × R₀^指数
```

**参数配置：**

| 时间段 | tanh 系数 | 指数 | 说明 |
|-------|----------|------|------|
| 7D | 0.12 | 1.8 | 短期波动大，使用更平滑的曲线 |
| 30D | 0.22 | 1.6 | 中期参数适中 |
| 90D | 0.18 | 1.6 | 长期数据更稳定 |

**参数选择依据：**
- tanh 系数控制曲线的陡峭程度，较大的系数使中等收益的差异更明显
- 指数控制高分区域的压缩程度，较小的指数使高收益者之间的差距更小
- 7D 使用较高的指数(1.8)是因为短期 ROI 波动大，需要更强的压缩

### 2. 回撤分 (0-8 分)

回撤分惩罚高风险交易者，最大回撤越小，分数越高。

**计算公式：**

```
回撤分 = 8 × max(0, 1 - |MDD| / 阈值)
```

**阈值配置：**

| 时间段 | 回撤阈值 | 说明 |
|-------|---------|------|
| 7D | 15% | 短期允许较小回撤 |
| 30D | 30% | 中期允许中等回撤 |
| 90D | 40% | 长期允许较大回撤 |

**设计逻辑：**
- 回撤超过阈值时，回撤分为 0
- 零回撤获得满分 8 分
- 线性插值计算中间值

### 3. 稳定分 (0-7 分)

稳定分基于胜率，奖励稳定盈利的交易者。

**计算公式：**

```
稳定分 = 7 × max(0, min(1, (胜率 - 45) / (上限 - 45)))
```

**阈值配置：**

| 时间段 | 基准胜率 | 上限胜率 | 说明 |
|-------|---------|---------|------|
| 7D | 45% | 62% | 短期胜率波动大 |
| 30D | 45% | 68% | 中期更稳定 |
| 90D | 45% | 70% | 长期最稳定 |

**设计逻辑：**
- 胜率低于 45% 时，稳定分为 0
- 胜率达到上限时，稳定分为 7
- 线性插值计算中间值

## 入榜门槛

为过滤低交易量和虚假账户，设置了 PnL 门槛：

| 时间段 | PnL 门槛 | 说明 |
|-------|---------|------|
| 7D | $300 | 周均 $42+ |
| 30D | $1,000 | 月均 $33+ |
| 90D | $3,000 | 日均 $33+ |

**未达门槛的交易员将不会显示 Arena Score。**

## 排名规则

当多个交易员分数相同时，按以下顺序排序：

1. Arena Score 降序（主排序）
2. 最大回撤绝对值升序（回撤小的优先）
3. 胜率降序（胜率高的优先）
4. PnL 降序（盈利多的优先）
5. 交易员 ID 字母序（确保稳定性）

## 总体分数计算（个人主页）

对于交易员个人主页，使用加权平均计算总体分数：

```
总体分数 = 0.70 × 90D分数 + 0.25 × 30D分数 + 0.05 × 7D分数
```

如果缺少某个时间段的数据，会进行相应的降权处理：

| 数据情况 | 计算方式 | 惩罚 |
|---------|---------|------|
| 完整数据 | 标准加权 | 无 |
| 缺少 90D | (0.80 × 30D + 0.20 × 7D) × 0.85 | 15% |
| 只有 7D | 7D × 0.70 | 30% |
| 只有 90D | 90D × 0.90 | 10% |

## 算法限制与说明

### 已知限制

1. **数据依赖性**：评分依赖交易所公开数据，不同交易所的数据完整性不同
2. **时间窗口**：短期分数波动大，不代表长期表现
3. **策略偏向**：算法偏向稳健型交易者，高波动策略可能被低估

### 特殊处理

- **Bybit 数据**：Bybit 的 PnL 是跟单者盈亏，不是交易员自身盈亏，因此跳过 PnL 门槛检查
- **GMX 数据**：GMX 只有 7D 和 30D 数据，不参与 90D 排名
