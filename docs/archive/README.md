# Documentation Archive

This directory contains historical documentation that has been consolidated into current documents.

## Why Archive?

These documents served important purposes during specific phases of the project, but have now been merged into comprehensive documents to:
- Reduce documentation fragmentation
- Make it easier to find information
- Maintain historical record without cluttering active docs

## Current Active Documents

Instead of reading archived documents, refer to:
- **docs/OPTIMIZATION_HISTORY.md** - Consolidated history of all optimization efforts
- **docs/AUDIT_HISTORY.md** - Consolidated history of all audits and reviews

## Archived Documents

### Optimization Reports
- `OPTIMIZATION_REPORT.md` - General technical optimization report
- `OPTIMIZATION_SUMMARY_2026-01.md` - 30-day optimization plan
- `PHASE1_CLEANUP_REPORT.md` - Phase 1 code cleanup

### Audit Reports
- `ARENA_COMMUNITY_AUDIT_REPORT.md` - Community and product audit
- `AUDIT_REPORT_2026-01-21.md` - State management and API audit
- `I18N_HARDCODE_AUDIT.md` - Internationalization hardcode audit
- `SUPABASE_SCHEMA_AUDIT.md` - Database schema audit

## Restoration

All documents are preserved in git history. To view original documents:

```bash
git log --all --full-history -- docs/archive/[filename]
```
