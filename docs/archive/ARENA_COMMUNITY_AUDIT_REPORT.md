## 一、冷启动失败风险分析

### 榜单空置风险

| 断点 | 触发条件 | 严重性 |
|------|----------|--------|
| **PnL 门槛过滤** | 7D 要求 >$300，30D >$1000，90D >$3000。新交易员难入榜 | 🔴 高 |
| **数据新鲜度** | 只返回 24h 内数据，cron 失败则全空 | 🔴 高 |
| **交易所单点故障** | GMX 无 90D 数据，新交易所接入缓慢 | 🟡 中 |

**空状态处理现状**：
- 显示 "暂无交易者数据" 静态文字
- 无引导用户做什么的 CTA
- 无展示"即将上榜"的预备队

### 社区冷启动断点

| 断点 | 当前状态 | 风险 |
|------|----------|------|
| **群组空帖** | 新建群组 0 帖子显示 "暂无帖子" | 🔴 用户流失 |
| **关注列表空** | "暂未关注任何人" 无推荐 | 🔴 无法形成社交图谱 |
| **热榜空置** | 依赖 `hot_score` 算法，冷启动时无热帖 | 🟡 内容发现失效 |
| **评论区冷** | "暂无评论，来发表第一条评论吧" 但无激励 | 🟡 互动不起来 |

### 最小修复手段

1. **种子内容预埋**：预置 10-20 个官方运营的高质量帖子
2. **榜单降级策略**：当 PnL 门槛过滤后少于 20 人时，自动降低门槛显示 "观察榜"
3. **空状态转化**：空列表时展示 "推荐关注 TOP5" 或 "热门群组"
4. **首发评论奖励**：第一条评论给予 Pro 徽章曝光加权

## 二、社区内容劣化路径审计

### 劣化风险路径

```
阶段1: 正常内容
    ↓
阶段2: 交易员晒单 → 引发跟单喊单
    ↓
阶段3: 小群主推荐付费群/信号 → 广告泛滥
    ↓
阶段4: 亏损用户情绪发泄 → 垃圾内容
    ↓
阶段5: 优质用户流失 → 社区死亡
```

### 当前防护能力

| 防护层 | 已实现 | 缺失 |
|--------|--------|------|
| **举报机制** | ✅ 分类举报(spam/harassment/misinformation) | ❌ 无自动检测 |
| **管理员审核** | ✅ 后台 ReportsTab | ❌ 无 AI 辅助 |
| **群主治理** | ✅ 民主投诉→投票罢免 (>100人群) | ❌ 小群无约束 |
| **禁言功能** | ✅ 管理员可禁言成员 | ❌ 无自动禁言 |

### 缺失的关键防护

1. **无敏感词过滤**：喊单、带单、私聊、WX、Telegram 等关键词无拦截
2. **无发帖频率限制**：可无限刷帖
3. **无链接/二维码检测**：外链和引流图无限制
4. **无新用户冷却期**：注册即可发帖
5. **无内容质量评分**：低质量内容无降权

### 需要提前限制的功能

| 功能 | 当前状态 | 建议限制 |
|------|----------|----------|
| 发帖 | 无限制 | 新用户 24h 后可发帖，每日上限 10 帖 |
| 私信 | 无限制 | 关注后才能私信，防骚扰 |
| 建群 | 无限制 | 账号满 7 天 + 3 帖子 |
| 评论带链接 | 允许 | 禁止或需审核 |

## 三、Pro 订阅价值评估 (付费用户视角)

### 定价概览

| 计划 | 原价 | 现价 | 折算月费 |
|------|------|------|----------|
| 月付 | $15 | $9.9 | $9.9 |
| 年付 | $180 | $99.9 | $8.3 |

### 权益价值分析

| 权益 | 价值判断 | 理由 |
|------|----------|------|
| **分类排行** | 🟡 低价值 | 数据本身不变，只是筛选视图 |
| **交易员变动提醒** | 🟢 有价值 | 但需要实现质量，当前只是 DM |
| **Arena Score 子分数** | 🟡 低价值 | 看总分已足够决策 |
| **高级筛选** | 🟡 低价值 | 用户很少用复杂筛选 |
| **交易员对比** | 🟢 有价值 | 对比功能刚需，但限制 5 人/次太少 |
| **Pro 徽章** | 🔴 不值钱 | 虚荣属性，无实际价值 |
| **Pro 专属群** | 🟡 待验证 | 取决于群内内容质量 |
| **历史数据 90D** | 🟢 有价值 | 免费只有 7D 太短 |
| **API 访问** | 🟢 高价值 | 对量化用户刚需 |
| **数据导出** | 🟢 有价值 | 但 10 次/月太少 |

### 不值钱的权益

1. **Pro 徽章**：无社交货币价值，无人在乎
2. **分类排行**：按类型筛选本应免费
3. **Arena Score 子分数**：过于专业，普通用户不理解百分位

### 必须增强的点

| 权益 | 当前 | 建议 |
|------|------|------|
| 交易员对比 | 5 人/次，10 次/月 | 10 人/次，无限次 |
| 数据导出 | 10 次/月 | 50 次/月 或无限 |
| 实时提醒 | 站内 DM | 增加 Email/Telegram/Webhook |
| 历史数据 | 90D | 增加 180D/365D |
| **缺失** | - | 交易员深度分析报告 (AI 生成) |
| **缺失** | - | 跟单模拟回测 |
| **缺失** | - | 风险预警阈值自定义 |

### 付费转化障碍

1. **无免费试用**：用户无法体验就付费
2. **无退出壁垒**：取消订阅无任何挽留
3. **Pro 群价值未体现**：500+ Pro 会员数据可疑（页面写死的假数据）

## 四、数据可信度自证能力检查

### "你这个数据是真的吗？" 用户质疑应对

| 质疑点 | 能否自证 | 缺失证据 |
|--------|----------|----------|
| **数据来源** | ✅ 显示来源 badge | ❌ 无交易所官方背书 |
| **更新时间** | ❌ 前端不显示 | 需加 "数据更新于 2h 前" |
| **原始链接** | ✅ 有 "查看原网页" | ✅ 可追溯 |
| **数据准确性** | ❌ 无法证明 | 无第三方审计、无校验和 |
| **Arena Score 公式** | ✅ 有文档 | 但用户不会看 docs/ |

### 无法自证的地方

1. **数据抓取是否完整**
   - 当前只展示 top 100
   - 用户无法知道是否有遗漏

2. **数据是否被篡改**
   - 无区块链存证
   - 无第三方审计

3. **历史数据保真**
   - 只有 `captured_at` 时间戳
   - 无历史快照对比

4. **交易员身份验证**
   - 头像/昵称从交易所抓取
   - 无法证明是同一人

5. **ROI 计算方式**
   - 各交易所计算逻辑不同
   - Arena 未统一说明

### 需要补的证据链

| 证据 | 实现难度 | 优先级 |
|------|----------|--------|
| 前端显示 "数据更新时间" | 低 | 🔴 P0 |
| 数据来源声明 footer | 低 | 🔴 P0 |
| Arena Score 计算说明 (简版) | 低 | 🟡 P1 |
| 交易所 API 截图存档 | 中 | 🟡 P1 |
| 数据差异说明 (各所 ROI 定义不同) | 低 | 🟡 P1 |
| 第三方审计报告 | 高 | 🟢 P2 |

## 六、用户误解点识别

### 用户一定会误解的地方

| 误解点 | 用户理解 | 实际情况 | 后果 |
|--------|----------|----------|------|
| **Arena Score** | 越高越好，100 分满分 | 综合评分，70+ 已是 top 级别 | 用户误以为 60 分是差的 |
| **ROI** | 统一计算标准 | 各交易所定义不同 | 跨所对比无意义 |
| **"500+ Pro 会员"** | 真实数据 | 硬编码的假数据 | 失去信任 |
| **关注交易员** | 会收到推送 | 只是列表收藏，无主动通知 (除非 Pro) | 功能期望落空 |
| **Max Drawdown** | 越低越安全 | 不同时间段计算基数不同 | 误判风险 |
| **排行榜实时性** | 实时更新 | 最快 15 分钟刷新 (热门) / 4 小时 (普通) | 错过最佳跟单时机 |

### 误解的严重后果

1. **Arena Score 误判**
   - 用户可能放弃 65 分的优质交易员
   - 追捧 90+ 分但实际风险高的交易员

2. **ROI 跨所对比**
   - Binance ROI 90D 和 Bybit ROI 90D 计算方式不同
   - 用户直接比数字，做出错误决策

3. **假数据曝光**
   - "500+ Pro 会员" 是 `pricing/page.tsx:609` 硬编码
   - 一旦被发现，品牌信任归零

## 七、对外叙事一致性检查

### 页面叙事对比

| 页面 | 核心叙事 | 问题 |
|------|----------|------|
| **首页** | 交易员排行榜 + 社区 | 社区入口弱，首屏无帖子 |
| **排行榜** | Arena Score 评分系统 | Score 说明不足，用户不懂 |
| **交易员主页** | 数据透明 + 一键跟单 | 无跟单入口，只能看 |
| **Pro 页面** | 风险管理专业工具 | 实际权益偏向数据筛选 |
| **定价页** | "500+ Pro 会员信任" | 假数据，与信任叙事矛盾 |

### 叙事冲突点

1. **"数据透明" vs 无数据更新时间**
   - 首页强调透明
   - 但用户看不到数据是何时更新的

2. **"一站式平台" vs 无跟单功能**
   - 交易员主页有 "关注" 但无 "跟单"
   - 用户需跳转交易所操作

3. **"社区驱动" vs 社区入口弱**
   - 首页侧边栏有帖子 feed
   - 但桌面端才能看到，移动端隐藏

4. **"专业工具" vs 权益同质化**
   - Pro 强调专业
   - 但权益都是筛选/对比，无专业分析报告

5. **"500+ Pro 会员" vs 无验证**
   - 定价页大字展示
   - 代码里是硬编码假数据

### 修复建议

| 冲突 | 修复方案 |
|------|----------|
| 数据透明 | 添加 "数据更新于 X 分钟前" |
| 无跟单 | 添加 "前往交易所跟单" 按钮，或明确定位为 "选单" 而非跟单 |
| 社区弱 | 首页增加社区 Tab，移动端底部导航加入社区 |
| Pro 叙事 | 增加 AI 分析报告、风险预警等专业功能 |
| 假数据 | 用 API 获取真实 Pro 用户数，或删除该展示 |
