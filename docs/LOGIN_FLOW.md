# 登录/注册流程说明

## 功能特性

### 1. 验证码注册（OTP）
- ✅ 发送真实验证码到邮箱
- ✅ 60秒倒计时，结束后可重发
- ✅ 验证码验证后设置密码和用户名
- ✅ 用户名唯一性检查

### 2. 密码注册
- ✅ 直接使用邮箱和密码注册

### 3. 密码登录
- ✅ 使用邮箱和密码登录

### 4. 邮箱链接登录（Magic Link）
- ✅ 发送登录链接到邮箱
- ✅ 点击链接自动登录
- ✅ 支持生产环境（不只是localhost）

## 使用流程

### 注册流程（验证码方式）

1. **输入邮箱**
   - 用户输入邮箱地址

2. **发送验证码**
   - 点击"发送验证码"按钮
   - 系统发送6位数字验证码到邮箱
   - 开始60秒倒计时
   - 倒计时期间按钮禁用

3. **输入验证码**
   - 用户输入收到的验证码
   - 点击"验证验证码"按钮

4. **设置用户名和密码**
   - 验证成功后，输入用户名（至少3个字符）
   - 输入密码（至少6个字符）
   - 系统检查用户名是否已存在
   - 点击"完成注册"完成注册

### 登录流程（邮箱链接）

1. **输入邮箱**
   - 在登录页面输入邮箱

2. **发送登录链接**
   - 点击"发送登录链接"按钮
   - 系统发送包含登录链接的邮件
   - 开始60秒倒计时

3. **点击邮件中的链接**
   - 用户打开邮箱，点击登录链接
   - 自动跳转到网站并完成登录

## Supabase 配置要求

### 1. 邮箱设置
- **Enable email confirmations**: 关闭（OTP不需要）
- **Enable Email provider**: 已启用

### 2. 重定向URL配置
在 Supabase Dashboard → Authentication → URL Configuration 中：

- **Site URL**: 你的网站URL（例如：`https://yourdomain.com`）
- **Redirect URLs**: 添加以下URL：
  - `https://yourdomain.com/login`
  - `http://localhost:3000/login`（开发环境）

### 3. 邮箱模板
- 使用默认的 Magic Link 模板
- 或自定义模板，确保包含验证码或登录链接

## 环境变量

确保以下环境变量已设置：

```bash
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

## 测试步骤

### 1. 测试验证码注册
1. 访问 `/login`
2. 点击"还没有账号？使用验证码注册"
3. 输入真实邮箱
4. 点击"发送验证码"
5. 检查邮箱中的验证码（6位数字）
6. 输入验证码并验证
7. 设置用户名和密码
8. 完成注册

### 2. 测试邮箱链接登录
1. 访问 `/login`
2. 输入已注册的邮箱
3. 点击"发送登录链接"
4. 检查邮箱中的登录链接
5. 点击链接，应该自动登录并跳转到首页

### 3. 测试密码登录
1. 访问 `/login`
2. 输入邮箱和密码
3. 点击"登录"
4. 应该成功登录并跳转到首页

## 注意事项

1. **生产环境配置**
   - 确保在 Supabase Dashboard 中配置了正确的 Site URL 和 Redirect URLs
   - 验证码和登录链接会发送到用户邮箱，确保邮箱服务正常工作

2. **倒计时功能**
   - 验证码和登录链接都有60秒倒计时
   - 倒计时期间不能重复发送
   - 倒计时结束后可以重新发送

3. **用户名唯一性**
   - 系统会检查用户名是否已被使用
   - 如果用户名已存在，会提示用户选择其他用户名

4. **Magic Link 处理**
   - 系统会自动检测URL中的 `access_token` 和 `type=magiclink`
   - 自动完成登录并跳转到首页
