## What Went Well

1. **Found the P0 blocker nobody noticed for 12 days** — `leaderboard_ranks` partition table missing `DEFAULT nextval()` on id column. Every compute-leaderboard upsert silently failed with NOT NULL violation. The cron reported "success" with 0 records processed. Homepage was completely empty.

2. **Root cause methodology** — instead of patching symptoms, traced each "half-product" complaint to its data source:
   - "0 followers" → compute-leaderboard hardcoded `followers: 0` instead of passing exchange data
   - "ugly avatars" → 51% of traders had identicon SVGs because DEX traders lack profile pics → fixed with exchange platform logos
   - "no disclaimer" → footer literally had no risk warning text

3. **Security audit caught real vulnerabilities**:
   - E2E test fixtures accessible in production (`?e2e_fixture=linked_accounts`)
   - API keys route leaking Supabase error messages (table names, constraints)
   - Subscription endpoint cached as `public` (sensitive user data)

4. **Performance wins from parallel audit**:
   - resolveTrader: serial 3-step waterfall → parallel Promise.all (saves 200-400ms)
   - Search: removed bio ILIKE without trigram index (was forcing seq scan)
   - Search: skip Supabase when Meilisearch available (saved wasted DB connection)
   - Similar traders: added Redis cache (15min TTL)

5. **TODO/FIXME cleaned 11→2** — all stale design token TODOs removed, remaining 2 are intentional (scheduled removal + HEIC support).

## What Could Be Better

1. **12 days undetected empty homepage** — pipeline health monitoring reported "success" because compute-leaderboard's upsert errors were logged but the cron still returned success status. **Action**: add a post-compute assertion that checks `COUNT(*) > 0` in leaderboard_ranks after upsert, fail the cron if empty.

2. **75% fix ratio** — highest ever. Good for a stabilization sprint, but indicates accumulated tech debt. **Action**: next session should be feature-focused to rebalance.

3. **Movers still empty** — Redis cached the empty result during the 12-day outage, and CDN cached that for 1 hour. Fixed TTLs (600→120s Redis, 3600→60s CDN for empty), but the root fix is the assertion from item 1.

4. **npm vulnerabilities still at 21** — remaining are all transitive (ws via viem/ethers/privy, @opentelemetry via trigger.dev). Need upstream releases. **Action**: check monthly for updates.
