## Recommendations (Priority Order)

### P0 - Fix Chinese-only user-facing text
1. Market components (CoreCards, MobileMarketTabs, TokenSidePanel, etc.)
2. ScoreBreakdown and ScoreRadar (visible on trader profiles)
3. WelcomeGuide (first thing new users see)
4. Legal pages (create bilingual versions)
5. Layout metadata (dynamic based on locale for SEO)

### P1 - Migrate inline patterns to t() system
Move `isZh ? '...' : '...'` patterns to use `t()` function for:
- Maintainability
- Single source of truth
- Easier future locale additions

### P2 - API error messages
Add bilingual support to API error responses by:
- Using the existing `lib/api/errors.ts` error code system consistently
- Passing locale header from client requests

### P3 - Utility/lib files
Add bilingual support to date formatting, validation messages, etc.
