平台基础
排行榜 & 交易员发现
整个平台的核心入口，用户来这里找到最好的交易员
▶
/ live
首页排行榜
多维度加权综合排名（ROI、胜率、Arena Score），支持按交易所筛选（VIP功能）、按时间段（7d/30d/全时）切换。使用 SSR 首屏渲染 + 无限滚动加载。数据来自 30+ 交易所的 trader_snapshots 表。
/trader/[handle] live
交易员详情页
展示单个交易员的完整画像：ROI/胜率/MDD/Sharpe/Sortino/PF、多时间段数据（7d/30d/全时）、权益曲线、持仓列表、历史交易记录。所有头像通过 /api/avatar 代理解决 CDN 防盗链问题。
/compare live
交易员对比
并排对比多个交易员的核心指标，支持雷达图可视化。适合用户在 follow 或复制交易前做决策。
/hot live
热门 / 趋势
基于近期活跃度和 Arena Score 增长率排序，展示当前最受关注的交易员。区别于综合排行榜，这里更强调"现在谁在涨"。
/exchange/[platform] live
按交易所浏览
每个交易所独立页面，展示该平台排行榜前列的交易员。数据来自 leaderboard_ranks 表。覆盖 Binance、OKX、Bybit、Bitget、Gate.io、Hyperliquid、GMX 等 30+ 交易所。
/search live
全局搜索
按交易员名称/handle/地址搜索，支持模糊匹配。还可以搜索 Library 内容、机构、工具。
社交层
消息 / 群组 / 动态 / 关注，Arena 从排行榜向社交交易平台演进的核心
▶
/messages live
私信系统
用户之间的一对一私信，实时消息（WebSocket 或 Supabase Realtime）。消息列表、对话详情、未读计数。支持文字、图片。
/groups live
群组
创建和加入交易主题社群，群组内可以发消息、分享交易策略。群组可以申请加入（/groups/[id]/apply），管理员审核后进入。
/post/[id] live
动态 / 帖子详情
用户发布的交易心得、市场分析、持仓分享。每个帖子独立页面，支持回复、点赞。
/my-posts live
我的动态
当前用户发布的所有帖子列表，可以管理和删除。
/following live
关注流
聚合关注的交易员和用户的最新动态，类似 Twitter 的 Home feed。显示交易员发布的内容、仓位更新、绩效变化。followers 数字显示的是平台内关注数，不是外部平台粉丝数。
/inbox live
通知 & 收件箱
系统通知（被关注、被评论、被提及）+ 私信预览。红色数字角标，不使用脉冲动画。
/notifications live
通知中心
所有平台通知的集中列表，可以按类型筛选（关注、评论、系统）、标记已读。
/tip live
打赏
用户可以向交易员或内容创作者打赏代币，奖励好的内容和策略分享。
/channels live
频道
类似 Telegram Channel，单向广播型内容流。交易员可以开设频道，关注者订阅接收。
Library 知识库
60,644 条内容，书籍 / 论文 / 白皮书 / 活动，全部站内阅读
▶
/library live
Library 主页
四个内容类型：Books（书籍）、Papers（学术论文）、Whitepapers（加密白皮书）、Events（行业活动）。按分类过滤、搜索标题/作者/标签。书籍封面通过 Google Books API 获取（4219 本有封面），没有封面则不显示，禁止任何自动生成的替代品。
/library/[id] live
内容详情 & 站内阅读
论文和白皮书已下载到 Supabase Storage（bucket: library-files），支持站内直接阅读，不跳外部链接。书籍 EPUB 仍在持续下载中（来源：Anna's Archive / LibGen，只能从 Mac Mini 抓取，VPS 被封了）。Papers 24,579 篇已有 PDF，Books 29,000+ 本 EPUB 覆盖率持续提升。
内容来源 & 限制
规则明确：不允许任何外部跳转，所有文件必须预下载到 Supabase Storage。书籍封面必须是出版社原版封面，没有就不显示。自动翻译的内容需要加"译"角标和颜色标记。
市场行情
Flash 快讯 / 市场数据 / 多链资产 / 治理追踪
▶
/flash-news live
Flash 快讯
9 个新闻源，每 2 分钟在 VPS（45.76.152.169）扫一遍，新内容推送到平台。实时更新，不刷新页面。这是平台的实时信息流核心。
/market live
市场数据
加密市场整体行情，主流资产价格、涨跌幅、成交量。提供宏观市场上下文，帮助用户理解排行榜交易员的近期表现。
/multi-chain live
多链视图
跨链资产追踪，聚合 ETH / Solana / Arbitrum / BSC 等主流链的数据，提供统一视角看多链市场状态。
/governance live
治理追踪
主要 DeFi 协议的治理投票动态，追踪重要提案。面向关注协议治理的研究者和持仓用户。
/kol live
KOL 榜
加密领域关键意见领袖榜单，追踪影响力和内容质量。与交易员排行榜不同，KOL 的排名依据是内容影响力，不是交易绩效。X/Twitter 数据对接选用 LunarCrush（$72-90/月），目前因预算暂缓。
/bot live
量化 Bot
平台自有量化机器人的展示和追踪页面，展示 bot 策略、历史绩效、当前持仓。

数据层 & 后端服务
VPS cron / LaunchAgent / enrichment / 30+ 交易所数据管道
▶
VPS 数据采集（45.76.152.169）
新加坡 VPS 运行所有交易所数据的定时采集 cron（每 1-3 小时），Flash 新闻（每 30 分钟）。路径 /opt/arena/scripts/，与 /opt/ranking-arena/ 是两个不同目录，push 之后两处都要同步。Puppeteer 在 VPS 上已损坏，Bybit/Bitget 等移到 Mac Mini 跑。
Mac Mini LaunchAgent
Mac Mini 上的 launchd 服务：daily-checkpoint.mjs（UTC 00:00，覆盖 OKX/Gains/Hyperliquid/Binance Futures 快照）、Bybit/Bitget Puppeteer 抓取（Bybit 需要 puppeteer+stealth 绕 WAF）、enrichment master loop 持续运行。
数据完整性状态
已完全清除：Binance Futures TC、Gains WR+TC、Jupiter Perps、XT、XT Spot。
无法解决：OKX Web3、Aevo（数据结构性缺失，确认后放弃）、dYdX、KuCoin。
进行中：WR null 每次 VPS cron 导入新交易员时约反弹 3-5K，master loop 持续补充中。
Sharpe Ratio：全平台 96-100% null，是数据完整性最大缺口。
API 代理层（/api/avatar、/api/...）
所有交易员头像通过 /api/avatar?url= 代理，解决各交易所 CDN 的 Referrer 防盗链问题。直接使用原 URL 会因 Referer 检查返回 403 导致头像无法加载。
Supabase 数据库架构
核心表：traders（基础信息）、trader_snapshots（带时间戳的绩效快照，包含 roi_7d/30d、win_rate、max_drawdown、sharpe_ratio 等）、leaderboard_ranks（交易所原始排名）、user_profiles（用户 + 绑定交易员）、library（知识库条目）。Pro 计划已升级，解决了 Free 计划无法创建大表索引的问题。
UI 设计规则
明确的不能做和必须做，防止返工
▶
绝对禁止
不允许任何 emoji 出现在 UI 中（文字/SVG/CSS 只能用几何符号 ● ▼ ▲）。
不允许 DiceBear / identicon / 自动生成的头像，没有原版头像就不显示。
不允许水平滚动，永远只能上下滑动（用户原话："永远让用户上下不要做左右滑动"）。
不允许写死深色颜色值，必须用 CSS 变量（支持亮/暗主题切换）。
不允许做辅助性优化（a11y lint SEO）直到核心数据完整性解决。
必须做到
通知角标：红色带数字，不脉冲不发光。
主题切换：View Transition API 圆形扩散动画（从按钮扩散）。
紫金渐变（purpleGold）：用于活跃分类/筛选按钮。
非 Pro 功能：明亮配色引导升级，不灰色隐藏。
自动翻译内容：加"译"角标 + 颜色标记。
交易员 ID 和头像：必须来自原平台，句柄纯数字的设为 NULL。
User = Trader：用户绑定交易所后，个人主页和交易员主页外观完全一致。
